tasarch::config::log_levels struct
#include <src/config/config.h>

This utility struct is used to hold the hierarchy of logging levels defined in a config.

Contents

  • Reference
    • Constructors, destructors, conversion operators
    • Public functions
    • Public variables

Lets take the following config for example:

[logging]
level = "info"
parent.level = "trace"
parent.child.level = "err"

The resulting struct could be initialized as follows:

log_levels root = {.level = info, .children = {
    {"parent": {.level = trace, .children = {
        "child": {.level = error, .children = {}}
    }}}
}};

auto tasarch::config::log_levels::get_level(const std::string& name) -> spdlog::level::level_enum

(recursively) scans hierarchy of logging levels, to find best match for name.

Parameters
name Hierarchical name of log level to retrieved, where each level in the hierarchy is separated by a ..
Returns spdlog::level::level_enum

For example, we have the following config:

[logging]
level = "info"
parent.level = "trace"
parent.child.level = "err"

A logger named parent.child will have a log level of err, one named parent.child2 will have trace. A logger named parent2 will have a log level of info
